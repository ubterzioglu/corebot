# Plan: Bot Kayıt Akışını `submissions` Tablosuna Yazacak Şekilde Güncelle

## Hedef
Bot'un Route 2 (Kayıt ol / Profil oluştur) akışı tamamlandığında, toplanan veriler `wa_users` yerine `submissions` tablosuna bir satır olarak yazılacak. `wa_users` tablosu yalnızca akış durumunu (`current_step`) ve `wa_id`'yi takip etmek için kalacak.

## Mevcut Durum
- Bot kayıt verilerini `wa_users` tablosuna yazıyor (adım adım update ile).
- `submissions` tablosu ayrı bir web formundan gelen veriler için kullanılıyor.
- Bot bir de Realtime subscription ile `submissions` tablosunu dinliyor (`whatsapp_interest=true` olan yeni satırlara WhatsApp mesajı gönderiyor).

## Değişiklik Planı

### 1. Akış durumunu `wa_users`'da tutmaya devam et
- `wa_users` tablosu sadece `wa_id`, `current_step`, `updated_at` ve akış sırasında geçici olarak toplanan verileri tutacak.
- Kayıt tamamlandığında (`ASK_PRIVACY_CONSENT` → onay), tüm kayıt verilerini `submissions` tablosuna insert et.

### 3. Kod değişiklikleri (`index.js`)

#### 3a. Yeni fonksiyon: `createSubmission(user)`
- Kayıt tamamlandığında (`privacy_consent = true`), `wa_users`'daki tüm toplanan verileri alıp `submissions` tablosuna insert eder.
- Insert başarılı olduktan sonra `wa_users`'da `registration_status`'u `"completed"` olarak günceller.

```js
async function createSubmission(user) {
  if (!supabase) return;

  const submission = {
    form_type: "whatsapp_bot",
    category: user.category,
    fullname: [user.name, user.surname].filter(Boolean).join(" ") || "Belirtilmedi",
    country: user.country || "Belirtilmedi",
    city: user.city || "Belirtilmedi",
    business: user.organization || null,
    field: user.occupation_interest || "Belirtilmedi",
    email: user.email,
    phone: user.phone || user.wa_id,
    description: null,
    referral_source: user.discovery_source || null,
    referral_code: user.referral_code || null,
    offers_needs: user.note || null,
    whatsapp_interest: !!user.whatsapp_group_interest,
    consent: true,
    status: "new",
    documents: [],
    contact_phone_reached: false,
    contact_whatsapp_reached: false,
    contact_instagram_reached: false,
    contact_email_reached: false,
  };

  const { data, error } = await supabase
    .from("submissions")
    .insert(submission)
    .select()
    .single();

  if (error) {
    console.error("submissions insert error:", error);
    throw error;
  }

  return data;
}
```

#### 3b. `ASK_PRIVACY_CONSENT` adımını güncelle
- Onaylandığında: `createSubmission(user)` çağır → ardından `wa_users`'ı güncelle (`registration_status: "completed"`, `registration_completed_at: now`).
- Reddedildiğinde: Mevcut davranış aynı kalır.

#### 3c. `getOrCreateUser` — gerekli alanları `wa_users`'dan çek

- `country` (NOT NULL): `ASK_COUNTRY` — mevcut hali zaten zorunlu.
- `city` (NOT NULL): `ASK_CITY` — mevcut hali zaten zorunlu.
- `field` (NOT NULL): `ASK_OCCUPATION_INTEREST` — mevcut hali zaten zorunlu.
- `email` (NOT NULL): `ASK_EMAIL` — mevcut hali zaten zorunlu.

- `documents` (NOT NULL): Sabit `[]`.

### 4. Realtime subscription kaldırılacak mı?
- Mevcut `processFormSubmission` ve `setupFormSubmissionSubscription` fonksiyonları web formundan gelen kayıtlar için çalışıyor.
- Bot'un kendisi `submissions`'a insert ettiğinde, Realtime listener bunu da yakalayıp **tekrar mesaj göndermeye çalışabilir** — bu bir döngüye neden olabilir.
- **Çözüm:** Realtime subscription filtresine `form_type=eq.whatsapp_bot` hariç tutma eklenebilir, VEYA `processFormSubmission` içinde `form_type === "whatsapp_bot"` olanları skip et.

### 5. `wa_users` tablosundaki gereksiz kolonlar
- Kayıt verileri artık `submissions`'da olduğu için, `wa_users`'dan bu kolonlar kaldırılabilir (bir sonraki adımda, bu plan kapsamında değil):
  - `name`, `surname`, `city`, `country`, `category`, `note`, `organization`, `occupation_interest`, `email`, `phone`, `discovery_source`, `referral_code`, `whatsapp_group_interest`, `privacy_consent`, `registration_status`, `registration_completed_at`, `funnel_interest`
- Ancak akış sırasında veriler geçici olarak `wa_users`'da tutuluyor. Insert tamamlandıktan sonra temizlenebilir veya kalabilir. **Bu kararı uygulamaya bırakıyorum — ilk sürümde `wa_users`'a yazmaya devam edip, kayıt tamamlanınca ek olarak `submissions`'a da yazacağız.**
