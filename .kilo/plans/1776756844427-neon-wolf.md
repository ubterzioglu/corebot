# WhatsApp Bot Flow Redesign — Implementation Plan

## Summary

Complete rewrite of the bot's conversational flow from the current linear 3-step onboarding (ASK_INTENT → ASK_CITY → DONE) to a menu-driven, skip-enabled, multi-path flow with 4 main routes: Hızlı Yönlendirme, Kayıt, Detaylı Form, İnsanla Görüş.

## Decisions

- **DB schema**: SQL migration file for new `wa_users` columns
- **Existing users**: Reset all to `WELCOME` step
- **Intent system**: Completely replace old 5-intent system with new 7-category system
- **Links**: Placeholder URLs via env vars

---

## Step 1: SQL Migration — New `wa_users` Columns

Create `supabase/migrations/001_bot_flow_v2.sql`:

**Add columns:**
- `name` text (nullable)
- `surname` text (nullable)
- `country` text (nullable)
- `category` text (nullable) — replaces `intent`
- `note` text (nullable)
- `funnel_interest` boolean (nullable)

**Drop old columns:**
- `intent` text
- `role` text

**Reset existing users:**
```sql
UPDATE wa_users SET current_step = 'WELCOME', intent = NULL, role = NULL;
```

**Update `current_step` enum** — new valid values:
`WELCOME`, `ASK_NAME`, `ASK_LOCATION`, `ASK_CATEGORY`, `ASK_NOTE`, `REDIRECT`, `DONE`, `REFERRAL_ASK`

## Step 2: Rewrite State Machine in `index.js`

### 2a. New Constants

**Env vars to add** (read from env, fallback to placeholder):
- `WA_CHANNEL_LINK` — WhatsApp channel invite
- `DETAILED_FORM_LINK` — detailed form URL
- `HUMAN_CONTACT_LINK` — defaults to `https://wa.me/491637084577`
- `WEBSITE_URL` — defaults to `https://corteqs.net/`

**MENU_TEXT constant** — reusable main menu:
```
1️⃣ Hızlı yönlendirme
2️⃣ Kayıt ol / Profil oluştur
3️⃣ Detaylı form (para kazanma & referral)
4️⃣ İnsanla görüş
```

**CATEGORY_MAP** — new 7-category system:
| # | Key | Label |
|---|-----|-------|
| 1 | career | İş & Kariyer |
| 2 | networking | Networking |
| 3 | relocation | Relokasyon |
| 4 | consulting | Danışmanlık |
| 5 | partnership | İş ortaklığı |
| 6 | monetization | Referral / Para kazanma |
| 7 | other | Diğer |

### 2b. New Helper Functions

Replace `normalizeIntent()` and `getIntentLabel()` with:

- **`parseMenuChoice(text)`** — detects "1"-"4" or keyword matching for main menu
- **`parseCategoryChoice(text)`** — detects "1"-"7" or keyword matching for category step
- **`isSkipWord(text)`** — returns true for "geç", "skip", "sonra", "hayır", "yok"
- **`isMenuWord(text)`** — returns true for "menü", "menu", "ana menü"
- **`parseName(text)`** — extracts {name, surname} from free text (1 word → name only, 2+ → first + last)
- **`parseLocation(text)`** — extracts {country, city} from free text (supports "Almanya - Dortmund" format and single-word input)

### 2c. Rewrite `buildReply(user, incomingText)`

New state machine — each `current_step` value maps to a handler:

#### WELCOME
- Any incoming message → show welcome + menu, set step to `MENU`
- If text is a menu choice (1-4) → go directly to that route

#### MENU
- Parse input:
  - `1` → skip to REDIRECT step
  - `2` → go to ASK_NAME step
  - `3` → show detailed form link + referral funnel
  - `4` → send human contact link
  - `isMenuWord` → re-show menu
  - unrecognized → AI fallback (re-show menu with guidance)

#### ASK_NAME
- `isMenuWord` → go to MENU
- `isSkipWord` → set name/surname null, go to ASK_LOCATION
- Otherwise → `parseName(text)`, save to user, go to ASK_LOCATION

#### ASK_LOCATION
- `isMenuWord` → go to MENU
- `isSkipWord` → set country/city null, go to ASK_CATEGORY
- Otherwise → `parseLocation(text)`, save to user, go to ASK_CATEGORY

#### ASK_CATEGORY
- `isMenuWord` → go to MENU
- `parseCategoryChoice(text)` → save category, go to ASK_NOTE
- Unrecognized → re-show category options

#### ASK_NOTE
- `isMenuWord` → go to MENU
- `isSkipWord` / "yok" → set note null, go to REDIRECT
- Otherwise → save note, go to REDIRECT

#### REDIRECT
- Show redirect options:
  ```
  1️⃣ Web sitesi → WEBSITE_URL
  2️⃣ WhatsApp kanalı → WA_CHANNEL_LINK
  3️⃣ İnsanla direkt iletişim → HUMAN_CONTACT_LINK
  4️⃣ Ana menüye dön
  ```
- Parse 1-4, send appropriate link, go to REFERRAL_ASK

#### REFERRAL_ASK
- Ask about detailed form interest:
  ```
  1️⃣ Evet, detaylı katılmak istiyorum → send DETAILED_FORM_LINK
  2️⃣ Hayır → "Sorun değil 👍"
  3️⃣ Daha sonra → "Sorun değil 👍"
  ```
- After response → go to DONE

#### DONE
- Any message → show "Ana menüye dönmek için 'menü' yaz."

### 2d. Global Input Pre-Processing

Before state machine logic, always check:
1. `isMenuWord(text)` → set step to `MENU`, return menu text (except when already in MENU step)
2. `text === "reset"` → same as `isMenuWord` (backward compat)

### 2e. Remove Old Code

- Delete `normalizeIntent()`
- Delete `getIntentLabel()`
- Delete `getTaskByIntent()`
- Delete `createTask()` calls from main flow (keep function for potential future use)
- Remove old step constants/logic for ASK_INTENT, ASK_CITY
