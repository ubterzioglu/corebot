# WhatsApp Bot Integration Plan for CorteQS Form Submissions

## Current State Analysis
- Existing WhatsApp bot (`index.js`) uses Express 5, handles user onboarding flow with Supabase `wa_users` table
- Current flow: ASK_INTENT → ASK_CITY → DONE with Turkish conversational interface
- Uses WhatsApp Business API via Meta Cloud API
- Has basic Supabase integration for `wa_users`, `wa_messages`, and `wa_tasks` tables
- Environment variables loaded from `.env` (gitignored) and `.secret` (tracked)

## Integration Requirements
Based on the CorteQS form submission documentation, we need to:
1. Monitor `submissions` table for new records where `whatsapp_interest = true`
2. Map submission categories to appropriate WhatsApp groups
3. Send personalized welcome messages with group invite links
4. Update submission status to prevent duplicate processing

## Implementation Strategy

### Option 1: Realtime Listener in Existing Bot (Recommended)
- Extend the current `index.js` bot to include a Supabase Realtime listener
- Use existing authenticated Supabase connection from service role key
- Process submissions as they arrive without needing separate infrastructure

### Option 2: Separate Edge Function
- Create a dedicated Supabase Edge Function that listens to `submissions` table
- Trigger WhatsApp messages via HTTP calls to the existing bot or directly to WhatsApp API
- More complex but better separation of concerns

**Decision**: Choose Option 1 (Realtime Listener) for simplicity and leveraging existing infrastructure.

## Detailed Implementation Steps

### Step 1: Add Category Mapping Configuration
- Define `GROUP_LINKS` constant mapping submission categories to WhatsApp group invite URLs
- Handle fallback categories (`bireysel` as default for unrecognized categories)

### Step 2: Implement Realtime Subscription
- Add Supabase Realtime channel subscription for `submissions` table
- Filter for `INSERT` events where `whatsapp_interest = true` and `status = 'new'`
- Use existing `supabase` client instance

### Step 3: Create WhatsApp Message Template
- Design welcome message template with personalization (name, city, country, group link)
- Ensure compliance with WhatsApp template messaging requirements

### Step 4: Implement Processing Logic
- Extract phone number (already in E.164 format from form)
- Determine appropriate group link based on category
- Send WhatsApp message using existing `sendWhatsAppMessage` function
- Update submission status to `'contacted'` to prevent reprocessing

### Step 5: Error Handling and Retry Logic
- Implement robust error handling for WhatsApp API failures
- Log errors appropriately without crashing the main bot
- Consider implementing retry mechanism for failed messages

### Step 6: Security and Rate Limiting
- Ensure proper authentication for database operations
- Implement rate limiting to respect WhatsApp API quotas
- Add validation to prevent spam/malicious submissions
